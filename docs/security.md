# 安全设计

## 1. 数据分类与脱敏

- 高敏：身份证、社保号、银行卡、完整证书编号、签章图、API Key。
- 敏感：姓名、电话、地址、投标报价、文件元数据。
- 普通：任务状态、规则代码、模板版本。

高敏编号默认只保留脱敏展示和 HMAC。原始 PDF、页面图、OCR 原文进入私有对象存储，
按项目授权访问。日志、ModelCallLog、AuditLog 和 Excel 不写未经授权的明文编号或
完整图片 Base64。当前三份历史样例 Excel 含明文身份证号，只作为离线兼容样本，
不得直接进入生产数据或对外分发。

## 2. 认证与授权

Phase B 实现 OIDC/OAuth2 或企业统一身份认证。RBAC 最小权限建议：

- administrator：用户、角色、系统配置。
- auditor：项目、审查、Finding 复核和报告。
- operator：上传、启动任务、查看处理状态。
- viewer：只读项目和已授权报告。

所有查询先做项目级授权；下载 URL 短时有效、单对象、可审计。人工复核和报告生成
属于高价值动作，记录 actor、request_id、资源版本和脱敏变更摘要。

## 3. 上传与内容安全

1. Nginx 和 API 双层限制大小，默认 500 MiB。
2. 校验扩展名、声明 MIME、文件魔数和 PDF 可解析性。
3. 文件先进入隔离区，调用 ClamAV/企业病毒扫描接口；通过后才转入正式桶。
4. 计算 SHA-256，按项目去重并创建不可变版本。
5. 限制 PDF 页数、解压后尺寸、渲染分辨率、OCR 时长，防止资源耗尽。
6. 解析器和 OCR Worker 使用低权限容器、只读根文件系统和独立临时目录。

当前代码只提供配置和数据字段，病毒扫描接口将在 Phase B 实现。

## 4. 模型调用安全

- Provider 凭据只从 Secret Manager/编排平台 Secret 注入。
- 调用前按任务最小化上下文并脱敏；需发送原图时要求项目明确授权和供应商合规评估。
- 禁止记录 API Key、原始高敏编号、完整请求图片和模型思考过程。
- 对输出做 Pydantic 校验、长度限制、风险措辞过滤和证据引用校验。
- Provider 超时、重试、熔断和配额分开配置；失败进入降级链。

## 5. 应用与基础设施

- TLS 在入口终止，内部生产网络按需启用 mTLS。
- PostgreSQL、Redis、MinIO 不暴露公网；对象桶默认私有。
- Nginx 按 IP 限流，应用层再按用户/项目/接口限流。
- 数据库账号分迁移、应用读写、只读报表；禁用超级用户运行应用。
- 生产禁用 FastAPI 文档页，设置安全响应头和可信代理列表。
- 依赖镜像固定版本并做 SCA、镜像扫描和 SBOM。

## 6. 保留、备份与响应

- 原始材料、报告和审计日志的保留期由项目合规策略决定，删除需双人审批和审计记录。
- PostgreSQL 每日全备 + WAL/PITR；MinIO 开启版本化和跨存储备份；Redis 不作为事实源。
- 每季度恢复演练，记录 RPO/RTO。疑似泄露时冻结下载、轮换密钥、保全审计证据并通知
  安全负责人。
